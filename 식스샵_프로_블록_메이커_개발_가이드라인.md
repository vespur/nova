1. 기본 개념
1.1 블록(Block)이란?

에디터를 통해 웹사이트 페이지에 삽입할 수 있는 디자인 단위
하나의 행(Row) 전체 또는 열(Column) 단위로 삽입 가능
여러 페이지, 여러 위치에서 재사용 가능하며 각각 독립적으로 설정 가능

1.2 블록 메이커(Block Maker)

블록을 제작하는 툴
HTML/CSS/JavaScript로 블록 코드 작성
세팅 빌더로 사용자 설정 패널 구성

1.3 컨텍스트(Context)

블록의 정보와 설정값을 담고 있는 JSON 객체
id: 블록의 고유 ID
property: 세팅 빌더로 정의한 설정값들
<data> 태그로 가져온 추가 데이터 (customer, cart, page 등)

2.3 <template> 태그

블록의 HTML 구조 정의
Handlebars.js 문법으로 동적 HTML 구성
블록 컨테이너 내부에 자동으로 렌더링됨

html<template>
  <div class="card">
    <h2>{{property.title}}</h2>
    
    {{#if property.showDescription}}
      <p>{{{property.description}}}</p>
    {{/if}}

      <div class="item">

</template>

## 4. 템플릿 제작 필수 요건

### 4.1 필수 조건 (미충족 시 등록 불가)

#### ✅ 편집 용이성
- [ ] 에디터에서 모든 기능이 오류 없이 작동
- [ ] 커스텀 블록이 정상 동작
- [ ] 사용자가 직접 편집 가능한 섹션/블록만 포함
- [ ] **통이미지 사용 금지** (글자가 이미지 안에 들어간 경우)
- [ ] 코드로 만든 결과물도 편집 가능해야 함

#### ✅ 저작권
- [ ] 모든 콘텐츠 및 리소스가 재사용 가능 (저작권 문제 없음)

#### ✅ 콘텐츠 구성
- [ ] 내 페이지 3개 이상 (서로 다른 레이아웃/내용)
- [ ] 헤더/푸터 메뉴 구성 완료
- [ ] 동일 페이지 반복 없음
- [ ] 무의미한 페이지 없음
- [ ] 활용도 높은 템플릿 구성 (단순 상품 진열 지양)

#### ✅ 이미지 품질
- [ ] 고품질 이미지 사용
- [ ] 해상도 낮은 이미지 없음
- [ ] 스톡 이미지 티 나는 과한 연출 지양
- [ ] 자연광 기반, 선명한 촬영
- [ ] 최신 트렌드에 맞는 톤앤매너

#### ✅ 일관성
- [ ] 이미지/동영상/아이콘의 디자인/스타일 일관성
- [ ] 더미 텍스트 없음
- [ ] 중복 이미지 없음
- [ ] 실제 예시 콘텐츠로 구성

- [ ] 내 페이지 5개 이상
