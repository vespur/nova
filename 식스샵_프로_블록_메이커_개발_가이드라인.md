## 1. 기본 개념

### 1.1 블록이란?

**블록(Block)**은 웹사이트를 구성하는 **재사용 가능한 콘텐츠 단위**입니다.

**예시**:
- Header 블록
- Hero Section 블록
- Feature Card 블록
- Footer 블록

각 블록은 **독립적으로 편집** 가능하며, 여러 페이지에서 **재사용** 가능합니다.

---

### 1.2 블록 메이커란?

**블록 메이커(Block Maker)**는 식스샵 프로에서 **커스텀 블록을 개발하는 도구**입니다.

**특징**:
- HTML, CSS, JavaScript로 블록 개발
- Handlebars.js 템플릿 엔진 사용
- 설정 패널(Setting Panel)로 비개발자도 쉽게 편집 가능

---

### 1.3 컨텍스트(Context)란?

**컨텍스트(Context)**는 블록에 전달되는 **데이터**입니다.

**예시**:
```javascript
{
  title: "노바파트너스",
  subtitle: "AI 솔루션 기업",
  buttonText: "문의하기",
  buttonLink: "/contact"
}
```

이 데이터를 블록 템플릿에서 `{{title}}`, `{{subtitle}}` 형태로 사용합니다.

## 2. 블록 메이커 핵심 문법

### 2.1 기본 구조

```html
<style>
  /* CSS 스타일 */
</style>

<template>
  <!-- HTML 템플릿 (Handlebars.js) -->
</template>

<script>
  // JavaScript 로직
</script>

<data>
  // 기본 데이터 (JSON)
</data>
```

---

### 2.2 `<style>` 태그

블록의 **CSS 스타일**을 정의합니다.

**예시**:
```html
<style>
  .hero {
    padding: 120px 0;
    background: linear-gradient(135deg, #0055FF 0%, #003399 100%);
  }
  
  .hero-title {
    font-size: 48px;
    font-weight: 700;
    color: white;
  }
</style>
```

**주의사항**:
- 클래스명은 **고유하게** 작성 (다른 블록과 충돌 방지)
- 예: `.hero-section-title` (O), `.title` (X - 너무 일반적)

---

### 2.3 `<template>` 태그

블록의 **HTML 구조**를 정의합니다.  
**Handlebars.js 문법**을 사용합니다.

**기본 예시**:
```html
<template>
  <section class="hero">
    <div class="container">
      <h1 class="hero-title">{{title}}</h1>
      <p class="hero-subtitle">{{subtitle}}</p>
      <a href="{{buttonLink}}" class="btn-primary">{{buttonText}}</a>
    </div>
  </section>
</template>
```

**조건문 예시**:
```html
{{#if showButton}}
  <a href="{{buttonLink}}" class="btn">{{buttonText}}</a>
{{/if}}
```

**반복문 예시**:
```html
{{#each features}}
  <div class="feature-card">
    <h3>{{this.title}}</h3>
    <p>{{this.description}}</p>
  </div>
{{/each}}
```

---

### 2.4 `<script>` 태그

블록의 **JavaScript 로직**을 정의합니다.

**예시**: 숫자 카운팅 애니메이션
```javascript
document.addEventListener('DOMContentLoaded', function() {
  const counters = document.querySelectorAll('.stat-number');
  
  counters.forEach(counter => {
    const target = parseInt(counter.getAttribute('data-target'));
    let count = 0;
    const increment = target / 100;
    
    const updateCount = () => {
      count += increment;
      if (count < target) {
        counter.textContent = Math.floor(count);
        requestAnimationFrame(updateCount);
      } else {
        counter.textContent = target;
      }
    };
    
    updateCount();
  });
});
```

### 2.5 `<data>` 태그

블록의 **기본 데이터**를 정의합니다.  
JSON 형식으로 작성합니다.

**예시**:
```json
{
  "title": "AI가 만드는 신뢰할 수 있는 미래",
  "subtitle": "금융, 제조, 의료, 소매 모든 산업에서 검증된 AI 솔루션",
  "buttonText": "도입 신청하기",
  "buttonLink": "/demo",
  "showButton": true,
  "features": [
    {
      "title": "빠른 도입",
      "description": "평균 6주 만에 운영 시작"
    },
    {
      "title": "검증된 신뢰성",

    }
  ]

```

## 3. 세팅 빌더 활용

### 3.1 세팅 빌더란?

**세팅 빌더(Setting Builder)**는 블록의 **설정 패널**을 정의하는 도구입니다.

비개발자가 블록을 쉽게 편집할 수 있도록 **GUI 설정 패널**을 만듭니다.

---

### 3.2 UX 전문가 관점의 설정 패널 설계 원칙

#### **핵심 철학**

> **"비개발자가 모든 디자인 요소를 직관적으로 제어할 수 있어야 한다"**

---

#### **정보 위계 (Information Hierarchy)**

설정 패널의 우선순위:

```
1순위: 콘텐츠 (무엇을 보여줄 것인가?)
    ↓
2순위: 디자인 (어떻게 보이는가?)
    ↓
3순위: 레이아웃 (어떻게 배치되는가?)
    ↓
4순위: 반응형 제어 (기기별 차이)
    ↓
5순위: 고급 옵션 (애니메이션, 특수효과)
```

---

#### **옵션 배치 3단계 구조**

**1단계: 공통 옵션 (데스크탑 + 모바일)**
- 텍스트 입력 필드 (데스크탑/모바일 함께 배치)
- 색상, 굵기 등 공통 스타일 옵션
- 사용자 시선 흐름 순서로 배치

**예시 순서**:
```
1. 제목 (가장 먼저 보이는 요소)
   - 데스크탑 제목 입력
   - 모바일 제목 입력
   - 제목 굵기
   - 제목 색상/그라데이션
   
2. 부제목 (두 번째로 보이는 요소)
   - 데스크탑 부제목 입력
   - 모바일 부제목 입력
   - 부제목 스타일 옵션
   
3. CTA 버튼 (행동 유도 요소)
   - 버튼 텍스트/링크
   - 버튼 디자인 옵션
   
4. 배경/미디어 (전체를 감싸는 요소)
   - 배경 타입 선택
   - 이미지/비디오 선택
```

**2단계: 데스크탑 전용 옵션**
- 섹션 전체 여백 (상/하/좌/우)
- 제목/부제목/본문 크기, 정렬, 간격
- 데스크탑 레이아웃 옵션

**3단계: 모바일 전용 옵션**
- 섹션 전체 여백 (상/하/좌/우)
- 제목/부제목/본문 크기, 정렬, 간격
- 표시/숨김 옵션
- 모바일 레이아웃 옵션

---

#### **수치 옵션 필수 규칙**

✅ **시작값**: 0부터 (특수한 경우 제외)  
✅ **증가 단위**: 1 (섬세한 조절 가능)  
✅ **최대값**: 넉넉하게 설정

**권장 범위표**:

| 옵션 타입 | Min | Max | Step | 단위 | 권장값 |
|-----------|-----|-----|------|------|--------|
| 폰트 크기 (데스크탑) | 0 | 100 | 1 | px | 제목 40-60px |
| 폰트 크기 (모바일) | 0 | 60 | 1 | px | 제목 28-36px |
| 섹션 여백 (상하) | 0 | 300 | 1 | px | 80-160px |
| 섹션 여백 (좌우) | 0 | 200 | 1 | px | 데스크탑 80px, 모바일 24px |
| 요소 간격 | 0 | 100 | 1 | px | 16-48px |
| 둥근 모서리 | 0 | 100 | 1 | px | 12-24px (2026 트렌드) |
| 투명도 | 0 | 100 | 1 | % | 80-95% |
| 그라데이션 각도 | 0 | 360 | 1 | deg | 135deg(대각선) |
| 애니메이션 속도 | 0.1 | 2.0 | 0.1 | 초 | 0.3-0.6초 |

#### **조건부 표시 (isVisible) 전략**

복잡도 감소를 위해 기본 옵션만 보여주고, 필요시 상세 옵션 표시:

```javascript
// Step 1: 기본 옵션
{
  "type": "CHECKBOX",
  "id": "showButton",
  "label": "버튼 표시",
  "default": true
}

// Step 2: 버튼 켜면 관련 옵션 표시
{
  "type": "TEXT",
  "id": "buttonText",
  "label": "버튼 텍스트",
  "isVisible": "{{showButton}}"
},

// Step 3: 고급 디자인 옵션도 조건부
{
  "type": "CHECKBOX",
  "id": "customizeButton",
  "label": "버튼 디자인 직접 설정",
  "default": false,
  "isVisible": "{{showButton}}",
  "help": "기본 스타일이 마음에 들지 않으면 체크하세요"
},
{
  "type": "COLOR_PICKER",
  "id": "buttonBackgroundColor",
  "label": "버튼 배경색",
  "isVisible": "{{and showButton customizeButton}}"
}
```

#### **좋은 설정 패널 체크리스트**

- [ ] 콘텐츠 → 디자인 → 레이아웃 → 반응형 순서
- [ ] 사용자의 시선 흐름대로 옵션 배치
- [ ] 관련 옵션끼리 그룹핑 (TITLE 활용)
- [ ] 텍스트 입력은 데스크탑/모바일 함께 배치
- [ ] 공통 옵션 먼저, 그 다음 데스크탑/모바일
- [ ] 모든 수치는 0부터 시작, 1단위 조절, 넉넉한 최대값
- [ ] 명확한 라벨 (이모지 활용)
- [ ] 도움말 제공 (help 속성)
- [ ] 조건부 표시로 복잡도 감소 (isVisible)
- [ ] 2026 트렌드 반영된 기본값

## 4. 템플릿 제작 필수 요건

### 4.1 편집 용이성

**목표**: 비개발자도 쉽게 편집 가능해야 합니다.

**체크리스트**:

### 8.3 JSON 일괄 붙여넣기 형식 (중요!)

**식스샵 프로의 설정 패널에서 "JSON으로 편집" 기능을 사용할 때 필수 형식입니다.**

#### **핵심 규칙**

```json
{
  "settings": [
    // 설정 항목들

  }
```

**✅ 필수 구조:**
1. **최상위에 `settings`와 `property` 두 개의 키**가 있어야 함
2. **`settings`**: 설정 패널의 UI 정의
3. **`property`**: 각 설정의 기본값 (초기값)

#### **올바른 예시**

```json
{

    {

    },
    {
      "type": "TEXT",
      "id": "title",
      "label": "제목",
      "placeholder": "제목을 입력하세요"
    },
    {
      "type": "RANGE",
      "id": "fontSize",
      "label": "글자 크기",
      "min": 12,
      "max": 48,
      "step": 2,

      "label": "아이콘 표시"

      "label": "아이콘 이미지",
```

#### **중요 포인트**

##### 1. **`property` 객체는 필수**
```json
// ❌ 작동하지 않음
{
  "settings": [...]
}

// ✅ 작동함
{
  "settings": [...],
  "property": {}
}
```

##### 2. **설정의 `id`와 `property` 키가 일치해야 함**
```json
{
  "settings": [
    {
      "id": "title",  // ← 이 id와
      "type": "TEXT"
    }
  ],
  "property": {
    "title": "기본값"  // ← 이 키가 일치
  }
}
```

##### 3. **LIST 타입의 property 구조**
```json
{
  "settings": [
    {
      "id": "items",
      "type": "LIST",
      "maxCount": 5,
      "settings": [
        {
          "id": "name",
          "type": "TEXT"
        },
        {
          "id": "value",
          "type": "TEXT"
        }
      ]
    }
  ],
  "property": {
    "items": [
      {
        "name": "항목1",
        "value": "값1"
      },
      {
        "name": "항목2",
        "value": "값2"
      }
    ]
  }
}
```

##### 4. **`isVisible` 문법**
```json
// ❌ 잘못된 문법
"isVisible": "{{property.showButton}}"

// ✅ 올바른 문법
"isVisible": "property.showButton === true"
```

**조건 연산자:**
- `===` : 같음
- `!==` : 다름
- `property.변수명` : 변수 참조
- `property.items[index].변수명` : LIST 내부 변수 참조

### 8.5 무한 루프 슬라이더 구현 예시

**사용 사례**: 고객사 로고 띠배너, 파트너 로고 슬라이더

#### **핵심 원리**

1. 로고를 **2번 복제**해서 배치
2. `translateX(-50%)`로 이동 → 끊김 없는 무한 반복
3. `animation-play-state`로 호버 시 멈춤

#### **CSS 구현**

```css
/* 컨테이너 */
.infinite-logo-slider {
  width: 100%;
  overflow: hidden;
  position: relative;
}

/* 슬라이더 트랙 (움직이는 부분) */
.logo-slider-track {
  display: flex;
  align-items: center;
  gap: 80px;
  width: fit-content;
  animation: scroll-logos 30s linear infinite;
}

/* 호버 시 멈춤 */
.infinite-logo-slider:hover .logo-slider-track {
  animation-play-state: paused;
}

/* 무한 루프 애니메이션 */
@keyframes scroll-logos {
  0% {
    transform: translateX(0);
  }
  100% {
    transform: translateX(-50%);  /* 정확히 절반 이동 */
  }
}

/* 로고 아이템 */
.logo-item {
  flex: 0 0 auto;
  height: 64px;
  padding: 0 16px;
}

.logo-item img {
  max-height: 100%;
  width: auto;
  object-fit: contain;
}
```

#### **HTML 구현** (Handlebars)

```html
<template>
  <div class="infinite-logo-slider">
    <div class="logo-slider-track">
      <!-- 원본 로고들 -->
      {{#each property.logos}}
      <div class="logo-item">
        <img src="{{this.image}}" alt="{{this.alt}}" />
      </div>
      {{/each}}
      
      <!-- 복제 로고들 (끊김 없는 무한 루프용) -->
      {{#each property.logos}}
      <div class="logo-item" aria-hidden="true">
        <img src="{{this.image}}" alt="" />
      </div>
      {{/each}}
    </div>
  </div>
</template>
```

#### **주요 포인트**

✅ **로고 2번 복제**: 끊김 없는 무한 반복을 위해 필수  
✅ **translateX(-50%)**: 정확히 절반 이동으로 매끄러운 반복  
✅ **aria-hidden="true"**: 복제본은 스크린리더에서 숨김 (접근성)  
✅ **linear**: 등속 운동으로 자연스러운 흐름  
✅ **animation-play-state**: 호버 시 일시정지 옵션

#### **성능 최적화**

```css
/* GPU 가속 활용 */
.logo-slider-track {
  will-change: transform;  /* 브라우저에 미리 알림 */
}

/* Reduced Motion 지원 */
@media (prefers-reduced-motion: reduce) {
  .logo-slider-track {
    animation: none !important;
  }
}
```

- [ ] **`isVisible` 문법 올바름** (중괄호 없이, `property.` 접두사)
- [ ] 통이미지 사용 없음
- [ ] 저작권 있는 이미지 사용 없음
- [ ] 반응형 디자인 완료
